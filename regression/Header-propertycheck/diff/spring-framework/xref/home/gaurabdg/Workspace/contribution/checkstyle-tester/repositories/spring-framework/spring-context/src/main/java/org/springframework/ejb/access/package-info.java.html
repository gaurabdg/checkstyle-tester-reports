1   /**
2    * This package contains classes that allow easy access to EJBs.
3    * The basis are AOP interceptors run before and after the EJB invocation.
4    * In particular, the classes in this package allow transparent access
5    * to stateless session beans (SLSBs) with local interfaces, avoiding
6    * the need for application code using them to use EJB-specific APIs
7    * and JNDI lookups, and work with business interfaces that could be
8    * implemented without using EJB. This provides a valuable decoupling
9    * of client (such as web components) and business objects (which may
10   * or may not be EJBs). This gives us the choice of introducing EJB
11   * into an application (or removing EJB from an application) without
12   * affecting code using business objects.
13   *
14   * <p>The motivation for the classes in this package are discussed in Chapter 11 of
15   * <a href="http://www.amazon.com/exec/obidos/tg/detail/-/0764543857/">Expert One-On-One J2EE Design and Development</a>
16   * by Rod Johnson (Wrox, 2002).
17   *
18   * <p>However, the implementation and naming of classes in this package has changed.
19   * It now uses FactoryBeans and AOP, rather than the custom bean definitions described in
20   * <i>Expert One-on-One J2EE</i>.
21   */
