2    * The classes in this package make JDBC easier to use and
3    * reduce the likelihood of common errors. In particular, they:
4    * <ul>
5    * <li>Simplify error handling, avoiding the need for try/catch/finally
6    * blocks in application code.
7    * <li>Present exceptions to application code in a generic hierarchy of
8    * unchecked exceptions, enabling applications to catch data access
9    * exceptions without being dependent on JDBC, and to ignore fatal
10   * exceptions there is no value in catching.
11   * <li>Allow the implementation of error handling to be modified
12   * to target different RDBMSes without introducing proprietary
13   * dependencies into application code.
14   * </ul>
15   *
16   * <p>This package and related packages are discussed in Chapter 9 of
