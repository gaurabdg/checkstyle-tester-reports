1   /**
2    * Theme support classes for Spring's web MVC framework.
3    * Provides standard ThemeResolver implementations,
4    * and a HandlerInterceptor for theme changes.
5    *
6    * <p>
7    * <ul>
8    * <li>If you don't provide a bean of one of these classes as {@code themeResolver},
9    * a {@code FixedThemeResolver} will be provided with the default theme name 'theme'.</li>
10   * <li>If you use a defined {@code FixedThemeResolver}, you will able to use another theme
11   * name for default, but the users will stick on this theme.</li>
12   * <li>With a {@code CookieThemeResolver} or {@code SessionThemeResolver}, you can allow
13   * the user to change his current theme.</li>
14   * <li>Generally, you will put in the themes resource bundles the paths of CSS files, images and HTML constructs.</li>
15   * <li>For retrieving themes data, you can either use the spring:theme tag in JSP or access via the
16   * {@code RequestContext} for other view technologies.</li>
17   * <li>The {@code pagedlist} demo application uses themes</li>
18   * </ul>
19   */
20  package org.springframework.web.servlet.theme;
