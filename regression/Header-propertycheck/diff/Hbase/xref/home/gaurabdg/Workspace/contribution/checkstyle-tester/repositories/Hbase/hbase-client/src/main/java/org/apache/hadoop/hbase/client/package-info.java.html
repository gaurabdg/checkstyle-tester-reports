29   <h2><a name="overview">Overview</a></h2>
30   <p>To administer HBase, create and drop tables, list and alter tables,
31   use {@link org.apache.hadoop.hbase.client.Admin}.  Once created, table access is via an instance
32   of {@link org.apache.hadoop.hbase.client.Table}.  You add content to a table a row at a time.  To
33   insert, create an instance of a {@link org.apache.hadoop.hbase.client.Put} object.  Specify value,
34   target column and optionally a timestamp.  Commit your update using
35   {@link org.apache.hadoop.hbase.client.Table#put(Put)}.
36   To fetch your inserted value, use {@link org.apache.hadoop.hbase.client.Get}.  The Get can be
37   specified to be broad -- get all on a particular row -- or narrow; i.e. return only a single cell
38   value.   After creating an instance of
39   Get, invoke {@link org.apache.hadoop.hbase.client.Table#get(Get)}.
40  
41   <p>Use {@link org.apache.hadoop.hbase.client.Scan} to set up a scanner -- a Cursor- like access.
42   After creating and configuring your Scan instance, call
43   {@link org.apache.hadoop.hbase.client.Table#getScanner(Scan)} and then
44   invoke next on the returned object.  Both {@link org.apache.hadoop.hbase.client.Table#get(Get)}
45   and {@link org.apache.hadoop.hbase.client.Table#getScanner(Scan)} return a
46  {@link org.apache.hadoop.hbase.client.Result}.
47  
48  <p>Use {@link org.apache.hadoop.hbase.client.Delete} to remove content.
49   You can remove individual cells or entire families, etc.  Pass it to
50   {@link org.apache.hadoop.hbase.client.Table#delete(Delete)} to execute.
51   </p>
52   <p>Puts, Gets and Deletes take out a lock on the target row for the duration of their operation.
53   Concurrent modifications to a single row are serialized.  Gets and scans run concurrently without
54   interference of the row locks and are guaranteed to not to return half written rows.
55   </p>
56   <p>Client code accessing a cluster finds the cluster by querying ZooKeeper.
57   This means that the ZooKeeper quorum to use must be on the client CLASSPATH.
58   Usually this means make sure the client can find your <code>hbase-site.xml</code>.
59   </p>
60  
61  <h2><a name="client_example">Example API Usage</a></h2>
62  
63  <p>Once you have a running HBase, you probably want a way to hook your application up to it.
64    If your application is in Java, then you should use the Java API. Here's an example of what
65    a simple client might look like.  This example assumes that you've created a table called
66    "myTable" with a column family called "myColumnFamily".
67  </p>
68  
69  <div style="background-color: #cccccc; padding: 2px">
70  <blockquote><pre>
71  import java.io.IOException;
72  
73  import org.apache.hadoop.hbase.HBaseConfiguration;
74  import org.apache.hadoop.hbase.TableName;
75  import org.apache.hadoop.hbase.client.Connection;
76  import org.apache.hadoop.hbase.client.ConnectionFactory;
77  import org.apache.hadoop.hbase.client.Get;
78  import org.apache.hadoop.hbase.client.Table;
79  import org.apache.hadoop.hbase.client.Put;
80  import org.apache.hadoop.hbase.client.Result;
81  import org.apache.hadoop.hbase.client.ResultScanner;
82  import org.apache.hadoop.hbase.client.Scan;
83  import org.apache.hadoop.hbase.util.Bytes;
84  
85  
86  // Class that has nothing but a main.
87  // Does a Put, Get and a Scan against an hbase table.
88  public class MyLittleHBaseClient {
89    public static void main(String[] args) throws IOException {
90      // You need a configuration object to tell the client where to connect.
91      // When you create a HBaseConfiguration, it reads in whatever you've set
92      // into your hbase-site.xml and in hbase-default.xml, as long as these can
93      // be found on the CLASSPATH
94      Configuration config = HBaseConfiguration.create();
95  
96      // Next you need a Connection to the cluster. Create one. When done with it,
97      // close it (Should start a try/finally after this creation so it gets closed
98      // for sure but leaving this out for readibility's sake).
99      Connection connection = ConnectionFactory.createConnection(config);
100     try {
101 
102       // This instantiates a Table object that connects you to
103       // the "myLittleHBaseTable" table (TableName.valueOf turns String into TableName instance).
104       // When done with it, close it (Should start a try/finally after this creation so it gets
105       // closed for sure but leaving this out for readibility's sake).
106       Table table = connection.getTable(TableName.valueOf("myLittleHBaseTable"));
107       try {
108 
109         // To add to a row, use Put.  A Put constructor takes the name of the row
110         // you want to insert into as a byte array.  In HBase, the Bytes class has
111         // utility for converting all kinds of java types to byte arrays.  In the
112         // below, we are converting the String "myLittleRow" into a byte array to
113         // use as a row key for our update. Once you have a Put instance, you can
114         // adorn it by setting the names of columns you want to update on the row,
115         // the timestamp to use in your update, etc.If no timestamp, the server
116         // applies current time to the edits.
117         Put p = new Put(Bytes.toBytes("myLittleRow"));
118 
119         // To set the value you'd like to update in the row 'myLittleRow', specify
120         // the column family, column qualifier, and value of the table cell you'd
121         // like to update.  The column family must already exist in your table
122         // schema.  The qualifier can be anything.  All must be specified as byte
123         // arrays as hbase is all about byte arrays.  Lets pretend the table
124         // 'myLittleHBaseTable' was created with a family 'myLittleFamily'.
125         p.add(Bytes.toBytes("myLittleFamily"), Bytes.toBytes("someQualifier"),
126         Bytes.toBytes("Some Value"));
127 
128         // Once you've adorned your Put instance with all the updates you want to
129         // make, to commit it do the following (The HTable#put method takes the
130         // Put instance you've been building and pushes the changes you made into
131         // hbase)
132         table.put(p);
133 
134         // Now, to retrieve the data we just wrote. The values that come back are
135         // Result instances. Generally, a Result is an object that will package up
136         // the hbase return into the form you find most palatable.
137         Get g = new Get(Bytes.toBytes("myLittleRow"));
138         Result r = table.get(g);
139         byte [] value = r.getValue(Bytes.toBytes("myLittleFamily"),
140           Bytes.toBytes("someQualifier"));
141         // If we convert the value bytes, we should get back 'Some Value', the
142         // value we inserted at this location.
143         String valueStr = Bytes.toString(value);
144         System.out.println("GET: " + valueStr);
145 
146         // Sometimes, you won't know the row you're looking for. In this case, you
147         // use a Scanner. This will give you cursor-like interface to the contents
148         // of the table.  To set up a Scanner, do like you did above making a Put
149         // and a Get, create a Scan.  Adorn it with column names, etc.
150         Scan s = new Scan();
151         s.addColumn(Bytes.toBytes("myLittleFamily"), Bytes.toBytes("someQualifier"));
152         ResultScanner scanner = table.getScanner(s);
153         try {
154            // Scanners return Result instances.
155            // Now, for the actual iteration. One way is to use a while loop like so:
156            for (Result rr = scanner.next(); rr != null; rr = scanner.next()) {
157              // print out the row we found and the columns we were looking for
158              System.out.println("Found row: " + rr);
159            }
160 
161            // The other approach is to use a foreach loop. Scanners are iterable!
162            // for (Result rr : scanner) {
163            //   System.out.println("Found row: " + rr);
164            // }
165          } finally {
166            // Make sure you close your scanners when you are done!
167            // Thats why we have it inside a try/finally clause
168            scanner.close();
169          }
170 
171          // Close your table and cluster connection.
172        } finally {
173          if (table != null) table.close();
174        }
175      } finally {
176        connection.close();
177      }
178   }
179 }
180 </pre></blockquote>
181 </div>
