19  /**
20   * <p>
21   * This package provides the definition and implementation of HBase's
22   * extensible data type API. {@link org.apache.hadoop.hbase.types.DataType}
23   * is the entry point. Basic type implementations are provided based on two
24   * different encoding schemes: the {@code Raw*} implementations use the
25   * {@code toXXX} methods in {@link org.apache.hadoop.hbase.util.Bytes} and
26   * the {@code Ordered*} implementations use the encoding scheme defined in
27   * {@link org.apache.hadoop.hbase.util.OrderedBytes}. Complex types are also
28   * supported in the form of {@link org.apache.hadoop.hbase.types.Struct} and
29   * the abstract {@code Union} classes.
30   * </p>
31   * <p>
32   * {@link org.apache.hadoop.hbase.types.DataType} implementations are used to
33   * convert a POJO into a {@code byte[]} while maintaining application-level
34   * constraints over the values produces and consumed. They also provide hints
35   * to consumers about the nature of encoded values as well as the relationship
36   * between different instances. See the class comments on
37   * {@link org.apache.hadoop.hbase.types.DataType} for details.
38   * </p>
39   * <p>
40   * The {@link org.apache.hadoop.hbase.types.DataType} interface is primarily
41   * of use for creating rowkeys and column qualifiers. It can also be used as a
42   * an encoder for primitive values. It does not support concerns of complex
43   * object serialization, concepts like schema version and migration. These
44   * concepts are handled more thoroughly by tools like Thrift, Avro, and
45   * Protobuf.
46   * </p>
47   *
48   * @since 0.95.2
49   */
50  package org.apache.hadoop.hbase.types;
