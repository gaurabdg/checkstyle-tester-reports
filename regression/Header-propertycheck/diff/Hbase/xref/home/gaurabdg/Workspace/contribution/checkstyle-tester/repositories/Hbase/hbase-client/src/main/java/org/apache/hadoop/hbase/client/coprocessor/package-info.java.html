20  /**
21  Provides client classes for invoking Coprocessor RPC protocols
22  
23  <p>
24  <ul>
25   <li><a href="#overview">Overview</a></li>
26   <li><a href="#usage">Example Usage</a></li>
27  </ul>
28  </p>
29  
30  <h2><a name="overview">Overview</a></h2>
31  <p>
32  The coprocessor framework provides a way for custom code to run in place on the
33  HBase region servers with each of a table's regions.  These client classes
34  enable applications to communicate with coprocessor instances via custom RPC
35  protocols.
36  </p>
37  
38  <p>
39  In order to provide a custom RPC protocol to clients, a coprocessor implementation
40  must:
41  <ul>
42   <li>Define a protocol buffer Service and supporting Message types for the RPC methods.
43   See the
44   <a href="https://developers.google.com/protocol-buffers/docs/proto#services">protocol buffer guide</a>
45   for more details on defining services.</li>
46   <li>Generate the Service and Message code using the protoc compiler</li>
47   <li>Implement the generated Service interface in your coprocessor class and implement the
48   {@link org.apache.hadoop.hbase.coprocessor.CoprocessorService} interface.  The
49   {@link org.apache.hadoop.hbase.coprocessor.CoprocessorService#getService()}
50   method should return a reference to the Endpoint's protocol buffer Service instance.
51  </ul>
52  Clients may then call the defined service methods on coprocessor instances via
53  the {@link org.apache.hadoop.hbase.client.Table#coprocessorService(byte[])},
54  {@link org.apache.hadoop.hbase.client.Table#coprocessorService(Class, byte[], byte[], org.apache.hadoop.hbase.client.coprocessor.Batch.Call)}, and
55  {@link org.apache.hadoop.hbase.client.Table#coprocessorService(Class, byte[], byte[], org.apache.hadoop.hbase.client.coprocessor.Batch.Call, org.apache.hadoop.hbase.client.coprocessor.Batch.Callback)}
56  methods.
57  </p>
58  
59  <p>
60  Since coprocessor Service instances are associated with individual regions within the table,
61  the client RPC calls must ultimately identify which regions should be used in the Service
62  method invocations.  Since regions are seldom handled directly in client code
63  and the region names may change over time, the coprocessor RPC calls use row keys
64  to identify which regions should be used for the method invocations.  Clients
65  can call coprocessor Service methods against either:
66  <ul>
67   <li><strong>a single region</strong> - calling
68     {@link org.apache.hadoop.hbase.client.Table#coprocessorService(byte[])}
69     with a single row key.  This returns a {@link org.apache.hadoop.hbase.ipc.CoprocessorRpcChannel}
70     instance which communicates with the region containing the given row key (even if the
71     row does not exist) as the RPC endpoint.  Clients can then use the {@code CoprocessorRpcChannel}
72     instance in creating a new Service stub to call RPC methods on the region's coprocessor.</li>
73   <li><strong>a range of regions</strong> - calling
74     {@link org.apache.hadoop.hbase.client.Table#coprocessorService(Class, byte[], byte[], org.apache.hadoop.hbase.client.coprocessor.Batch.Call)}
75     or {@link org.apache.hadoop.hbase.client.Table#coprocessorService(Class, byte[], byte[], org.apache.hadoop.hbase.client.coprocessor.Batch.Call, org.apache.hadoop.hbase.client.coprocessor.Batch.Callback)}
76     with a starting row key and an ending row key.  All regions in the table
77     from the region containing the start row key to the region containing the end
78     row key (inclusive), will we used as the RPC endpoints.</li>
79  </ul>
80  </p>
81  
82  <p><em>Note that the row keys passed as parameters to the <code>Table</code>
83  methods are not passed directly to the coprocessor Service implementations.
84  They are only used to identify the regions for endpoints of the remote calls.
85  </em></p>
86  
87  <p>
88  The {@link org.apache.hadoop.hbase.client.coprocessor.Batch} class defines two
89  interfaces used for coprocessor Service invocations against multiple regions.  Clients implement
90  {@link org.apache.hadoop.hbase.client.coprocessor.Batch.Call} to call methods of the actual
91  coprocessor Service instance.  The interface's <code>call()</code> method will be called once
92  per selected region, passing the Service instance for the region as a parameter.  Clients
93  can optionally implement {@link org.apache.hadoop.hbase.client.coprocessor.Batch.Callback}
94  to be notified of the results from each region invocation as they complete.
95  The instance's {@link org.apache.hadoop.hbase.client.coprocessor.Batch.Callback#update(byte[], byte[], Object)}
96  method will be called with the {@link org.apache.hadoop.hbase.client.coprocessor.Batch.Call#call(Object)}
97  return value from each region.
98  </p>
99  
100 <h2><a name="usage">Example usage</a></h2>
101 <p>
102 To start with, let's use a fictitious coprocessor, <code>RowCountEndpoint</code>
103 that counts the number of rows and key-values in each region where it is running.
104 For clients to query this information, the coprocessor defines the following protocol buffer
105 service:
106 </p>
107 
108 <div style="background-color: #cccccc; padding: 2px">
109 <blockquote><pre>
110 message CountRequest {
111 }
112 
113 message CountResponse {
114   required int64 count = 1 [default = 0];
115 }
116 
117 service RowCountService {
118   rpc getRowCount(CountRequest)
119     returns (CountResponse);
120   rpc getKeyValueCount(CountRequest)
121     returns (CountResponse);
122 }
123 </pre></blockquote></div>
124 
125 <p>
126 Next run the protoc compiler on the .proto file to generate Java code for the Service interface.
127 The generated {@code RowCountService} interface should look something like:
128 </p>
129 <div style="background-color: #cccccc; padding: 2px">
130 <blockquote><pre>
131 public static abstract class RowCountService
132   implements com.google.protobuf.Service {
133   ...
134   public interface Interface {
135     public abstract void getRowCount(
136         com.google.protobuf.RpcController controller,
137         org.apache.hadoop.hbase.coprocessor.example.generated.ExampleProtos.CountRequest request,
138         com.google.protobuf.RpcCallback<org.apache.hadoop.hbase.coprocessor.example.generated.ExampleProtos.CountResponse> done);
139 
140     public abstract void getKeyValueCount(
141         com.google.protobuf.RpcController controller,
142         org.apache.hadoop.hbase.coprocessor.example.generated.ExampleProtos.CountRequest request,
143         com.google.protobuf.RpcCallback<org.apache.hadoop.hbase.coprocessor.example.generated.ExampleProtos.CountResponse> done);
144   }
145 }
146 </pre></blockquote></div>
147 
148 <p>
149 Our coprocessor Service will need to implement this interface and the {@link org.apache.hadoop.hbase.coprocessor.CoprocessorService}
150 in order to be registered correctly as an endpoint.  For the sake of simplicity the server-side
151 implementation is omitted.  To see the implementing code, please see the
152 {@link org.apache.hadoop.hbase.coprocessor.example.RowCountEndpoint} class in the HBase source code.
153 </p>
154 
155 <p>
156 Now we need a way to access the results that <code>RowCountService</code>
157 is making available.  If we want to find the row count for all regions, we could
158 use:
159 </p>
160 
161 <div style="background-color: #cccccc; padding: 2px">
162 <blockquote><pre>
163 Connection connection = ConnectionFactory.createConnection(conf);
164 Table table = connection.getTable(TableName.valueOf("mytable"));
165 final ExampleProtos.CountRequest request = ExampleProtos.CountRequest.getDefaultInstance();
166 Map<byte[],Long> results = table.coprocessorService(
167     ExampleProtos.RowCountService.class, // the protocol interface we're invoking
168     null, null,                          // start and end row keys
169     new Batch.Call<ExampleProtos.RowCountService,Long>() {
170         public Long call(ExampleProtos.RowCountService counter) throws IOException {
171           BlockingRpcCallback<ExampleProtos.CountResponse> rpcCallback =
172               new BlockingRpcCallback<ExampleProtos.CountResponse>();
173           counter.getRowCount(null, request, rpcCallback);
174           ExampleProtos.CountResponse response = rpcCallback.get();
175           return response.hasCount() ? response.getCount() : 0;
176         }
177     });
178 </pre></blockquote></div>
179 
180 <p>
181 This will return a <code>java.util.Map</code> of the <code>counter.getRowCount()</code>
182 result for the <code>RowCountService</code> instance running in each region
183 of <code>mytable</code>, keyed by the region name.
184 </p>
185 
186 <p>
187 By implementing {@link org.apache.hadoop.hbase.client.coprocessor.Batch.Call}
188 as an anonymous class, we can invoke <code>RowCountService</code> methods
189 directly against the {@link org.apache.hadoop.hbase.client.coprocessor.Batch.Call#call(Object)}
190 method's argument.  Calling {@link org.apache.hadoop.hbase.client.Table#coprocessorService(Class, byte[], byte[], org.apache.hadoop.hbase.client.coprocessor.Batch.Call)}
191 will take care of invoking <code>Batch.Call.call()</code> against our anonymous class
192 with the <code>RowCountService</code> instance for each table region.
193 </p>
194 
195 <p>
196 Implementing {@link org.apache.hadoop.hbase.client.coprocessor.Batch.Call} also allows you to
197 perform additional processing against each region's Service instance.  For example, if you would
198 like to combine row count and key-value count for each region:
199 </p>
200 
201 <div style="background-color: #cccccc; padding: 2px">
202 <blockquote><pre>
203 Connection connection = ConnectionFactory.createConnection(conf);
204 Table table = connection.getTable(TableName.valueOf("mytable"));
205 // combine row count and kv count for region
206 final ExampleProtos.CountRequest request = ExampleProtos.CountRequest.getDefaultInstance();
207 Map<byte[],Long> results = table.coprocessorService(
208     ExampleProtos.RowCountService.class, // the protocol interface we're invoking
209     null, null,                          // start and end row keys
210     new Batch.Call<ExampleProtos.RowCountService,Pair<Long,Long>>() {
211        public Long call(ExampleProtos.RowCountService counter) throws IOException {
212          BlockingRpcCallback<ExampleProtos.CountResponse> rowCallback =
213              new BlockingRpcCallback<ExampleProtos.CountResponse>();
214          counter.getRowCount(null, request, rowCallback);
215 
216          BlockingRpcCallback<ExampleProtos.CountResponse> kvCallback =
217              new BlockingRpcCallback<ExampleProtos.CountResponse>();
218          counter.getKeyValueCount(null, request, kvCallback);
219 
220          ExampleProtos.CountResponse rowResponse = rowCallback.get();
221          ExampleProtos.CountResponse kvResponse = kvCallback.get();
222          return new Pair(rowResponse.hasCount() ? rowResponse.getCount() : 0,
223              kvResponse.hasCount() ? kvResponse.getCount() : 0);
224     }
225 });
226 </pre></blockquote></div>
227 */
228 package org.apache.hadoop.hbase.client.coprocessor;
