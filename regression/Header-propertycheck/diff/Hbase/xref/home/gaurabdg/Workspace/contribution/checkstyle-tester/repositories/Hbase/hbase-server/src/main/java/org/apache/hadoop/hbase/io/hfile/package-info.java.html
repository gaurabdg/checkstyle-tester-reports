18  /**
19   * Provides implementations of {@link org.apache.hadoop.hbase.io.hfile.HFile} and HFile
20   * {@link org.apache.hadoop.hbase.io.hfile.BlockCache}.  Caches are configured (and instantiated)
21   * by {@link org.apache.hadoop.hbase.io.hfile.CacheConfig}.  See head of the
22   * {@link org.apache.hadoop.hbase.io.hfile.CacheConfig} class for constants that define
23   * cache options and configuration keys to use setting cache options. Cache implementations
24   * include the default, native on-heap {@link org.apache.hadoop.hbase.io.hfile.LruBlockCache} and a
25   * {@link org.apache.hadoop.hbase.io.hfile.bucket.BucketCache} that has a bunch of deploy formats
26   * including acting as a L2 for LruBlockCache -- when a block is evicted from LruBlockCache, it
27   * goes to the BucketCache and when we search a block, we look in both places -- or, the
28   * most common deploy type,
29   * using {@link org.apache.hadoop.hbase.io.hfile.CombinedBlockCache}, BucketCache is used as
30   * a host for data blocks with meta blocks in an instance of LruBlockCache.  BucketCache
31   * can also be onheap, offheap, and file-backed.
32   * 
33   * <h1>Which BlockCache should I use?</h1>
34   * By default LruBlockCache is on.  If you would like to cache more, and offheap (offheap
35   * usually means less GC headache), try enabling * BucketCache. Fetching will always
36   * be slower when fetching from BucketCache but latencies tend to be less erratic over time
37   * (roughly because GC is less). See Nick Dimiduk's
38   * <a href="http://www.n10k.com/blog/blockcache-101/">BlockCache 101</a> for some numbers.
39   *
40   * <h1>Enabling {@link org.apache.hadoop.hbase.io.hfile.bucket.BucketCache}</h2>
41   * See the HBase Reference Guide <a href="http://hbase.apache.org/book.html#enable.bucketcache">Enable BucketCache</a>.
42   *
43   */
